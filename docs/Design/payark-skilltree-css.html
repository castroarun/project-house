Randomization Engine
Every Attempt Is Unique
↹

Random Selection

Each attempt pulls N questions from the pool using a unique seed. Never the same set twice.

⇅

Answer Shuffling

A/B/C/D positions are randomized every attempt. Same question, different answer order.

⚖

Weighted Retry

Questions you got wrong before have a higher chance of reappearing. Forces genuine learning.

▦

Category Coverage

Each attempt guarantees topic coverage (e.g., 15% Kafka, 20% DDD). No lucky easy subsets.

▲

Difficulty Spread

Every attempt: 30% easy, 50% medium, 20% hard. Maintained regardless of randomization.

Anti-Cheat Architecture
4 Defense Layers
Cheating is either impossible or self-defeating. The real exam is: can you build the system?
1

Structural

300-question pool for a 60-question exam. Encrypted questions in DB, no answer column stored alongside. Memorizing all 300 = learning the material anyway.

2

Temporal

Per-question timer (60-90s MCQ, 5 min scenario). No back-navigation during exam. 48-hour cooldown between reattempts. Prevents tab-switching and brute-force retry.

3

Feedback

Correct answers never shown after failure. Only category weakness breakdown: "Review Module M07" not "Answer was C". Forces re-study, not memorization.

4

Ultimate

Level 4 (PAKA) is a real project build. Cannot fake a working codebase + architecture document. Automated gates, AI code review, and written architecture evaluation.
